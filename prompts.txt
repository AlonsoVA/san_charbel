================================================================================
PROMPTS UTILIZADOS CON IA (Cursor) — Revisión y análisis Backend API
Notas de Evolución (san_charbel)
================================================================================

Nota:
Los siguientes prompts fueron utilizados como apoyo para revisar, validar y analizar
la estructura del backend desarrollado en Next.js con MongoDB y Mongoose.
El enfoque principal fue verificar compatibilidad, lógica, validaciones e integridad
del sistema ya implementado, no solicitar que la IA desarrollara completamente el proyecto.
En algunas partes la IA completó código.

--------------------------------------------------------------------------------
CONTEXTO GENERAL
--------------------------------------------------------------------------------

Se trabajó en un backend utilizando Next.js App Router, JavaScript, MongoDB y Mongoose
para un módulo médico de notas de evolución.

El sistema incluye:

- Pacientes con expediente y alergias
- Notas de evolución inmutables
- Notas aclaratorias asociadas a notas originales
- Alertas de alergias al momento de crear notas
- Endpoints API dentro de app/api
- Modelos Mongoose dentro de models/
- Uso de Mongoose 9 y MongoDB

La IA se utilizó principalmente para:

- Revisar la estructura implementada
- Detectar posibles errores
- Validar compatibilidad entre versiones
- Analizar lógica de inmutabilidad
- Revisar manejo de errores y validaciones

--------------------------------------------------------------------------------
PROMPT 1 — Revisión conexión MongoDB
--------------------------------------------------------------------------------

Revisa la implementación realizada en lib/mongodb.js.

Analiza si la conexión está correctamente estructurada para Next.js App Router y
Mongoose 9 utilizando un patrón singleton para evitar múltiples conexiones durante
el hot-reload en desarrollo.

Verifica:

- Uso correcto de MONGODB_URI
- Manejo de errores si la variable no existe
- Compatibilidad con Next.js y Mongoose 9
- Que no existan problemas de reconexión
- Que el patrón utilizado sea adecuado para desarrollo y producción

Indica posibles mejoras o riesgos encontrados en la estructura actual.
